While-løkker

En while-løkke fungerer litt som en for-løkke.
Den vil sjekke om noe er sant, utføre koden om det er sant, gå tilbake for å sjekke sannhetsverdien igjen, inntil den får et usant resultat.
Det som er anderledes er at while løkken ikke inkluderer en tellerverdi.
En while løkke har derfor ikke et like klart sluttpunkt.
Dette kan være praktisk hvis vi ikke vet hvor mange ganger vi skal gå gjennom en løkke.
Et eksempel på dette kan være om en bruker skal legge til linjer til en liste, og selv skal bestemme når de vil slutte.
Vi lager et slik eksempel. Begynn med et skjellet og importer biblioteket JOptionPane.

package pakke;

import static javax.swing.JOptionPane.*;

public class WhileLokker {

	public static void main(String[] args) {
		

	}

}

Vi lager to variabler, "liste" og "ferdig".

StringBuilder liste = new StringBuilder();
boolean ferdig = false;

"liste" er spesiel siden det er en ny type variabel, StringBuilder.
Navnet forklarer hva variabelen gjør ganske bra, den bygger opp en String verdi.
StringBuilder har en metode append(), som lar deg legge til hva som helst, forskjellige typer blir konvertert til typen String.
"ferdig" er også en ny type variabel. Den inneholder en sannhetsverdi, true eller false.
Siden if og løkker sjekker sannhetsverdier kan vi gi en variabel som refererer til en sannhetsverdi.

Nå kan vi begynne på while-løkken:

while(!ferdig) {

}

while-løkken ser mye enklere ut enn for-løkken.
Det er siden vi ikke teller hvor mange ganger vi har gått gjennom løkken nå.
Alt vi vil vite er om vi er ferdige. Om vi ikke er ferdige fortsetter vi.
Det er dette while-løkken spør om, !ferdig betyr "ikke ferdig", et utropstegn den motsatte sannhetsverdien.
Det vil si at vi går inn i while-løkken, siden ferdig == false.

Vi vil begynne å spør om linjer til listen nå.
Siden vi skal bruke Stringen vi får til å sjekke om brukeren er ferdig skal vi også nevne det i meldingen som sendes til brukeren.

String nyLinje = showInputDialog("Legg en linje til listen (gi et tomt svar for å slutte):");

Vi må huske å teste om de er ferdige før vi legger noe til listen.

if(!nyLinje.isEmpty()){


}

isEmpty() er en String metode som lar deg se om en String er tom(mellomrom telles ikke som tom).
Det er vanligvis sant at det ikke er tilfellet at nyLinje er tom(!nyLinje.isEmpty()).
Hvis dette er sant legger vi linjen til liste og bruker et spesielt "neste linje" tegn, "\n":

liste.append(nyLinje + "\n");

Merk at "neste linje" tegnet er innenfor hermetegn og bruker en mer uvanlig omvendt skråstrek.

Hvis det skulle være tilfellet at den nye linjen faktisk er tom, setter vi ferdig variabelen våres til true for å stoppe løkken.

else {

ferdig = true;

}

else her betyr alt annet enn (!nyLinje.isEmpty).
Det er ikke tilfellet at det ikke er tilfellet at den nye linjen er tom, altså er det tilfellet at den nye linjen er tom.
Det vil si at else betyr det samme som "else if(nyLinje.isEmpty())".

Etter while-løkken er ferdig vil vi skrive ut listen vi har laget. Her kan vi behandle liste som en vanlig String.

showMessageDialog(null, "Liste:\n" + liste);

Nå burde alt fungere og koden burde se omtrent slik ut:

package pakke;

import static javax.swing.JOptionPane.*;

public class WhileLokker {

	public static void main(String[] args) {
		
		StringBuilder liste = new StringBuilder();
		boolean ferdig = false;
		
		while(!ferdig) {
			
			String nyLinje = showInputDialog("Legg en linje til listen (gi et tomt svar for å slutte):");
			
			if (!nyLinje.isEmpty()) {
				
				liste.append(nyLinje + "\n");
				
			} else {
				
				ferdig = true;
				
			}
			
		}
		
		showMessageDialog(null, "Liste:\n" + liste);
	}

}
