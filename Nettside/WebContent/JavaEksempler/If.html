If

Til nå har vi laget programmer som følger de samme instruksjonene hver gang.
Programmet som leser inn et navn, og bruker det, kan få forskjellig informasjon, men det vil alltid bruke det på samme måte.
Hvis vi ser på noe annet, som innlogging, vet vi at det er tilfeller hvor vi må handle forskjellig basert på forskjellige resultater.
Dette gjør vi med if. Som et eksempel kan vi lage et system hvor du skal gjette et tall.
La oss lage et skjelett, importere JOptionPane og lese inn en verdi vi kaller tallTxt:

package pakke;

import static javax.swing.JOptionPane.*;

public class GjettTall { 

	public static void main(String[] args) {
	
		String tallTxt = showInputDialog("Gjett tallet:");
		
	}

}

Nå er det vi vil ha et tall, så vi bruker en int(heltall) variabel og en metode som kalles parseInt().
parseInt() tar inn en String og returnerer en int verdi.
Først må vi importere biblioteket som inneholder parseInt() ved å skrive:

import static java.lang.Interger.*;

Deretter lager vi en ny int. Den skal bli gitt verdien som parseInt(tallTxt) returnerer, selve tallet i String-verdien:

int tall = parseInt(tallTxt);

Dersom du synes det er oversiktlig nok for deg kan du også forkorte tallTxt og tall til:

int tall = parseInt(showInputDialog("Gjett tallet:"));

Vi vil lage en int hemmeligTall med et tilfeldig tall som skal gjettes.
Hvordan vil vi vite om vi gjettet det riktige tallet?
Vi vil bruke if for å sjekke dette.
Vi starter med å skrive dette i main metoden:

if(tallTxt == hemmeligTall){

}

Så, hva betyr dette? Vi starter med "if".
"if" spør om noe skjer, og hvis det skjer vil koden mellom krøllparentesene til if bli utført.
Her spør vi om tallTxt == hemmeligTall, men hvorfor bruker vi to er-lik-tegn?
Grunnen til at vi gjør det er at vi skiller mellom deklarasjon og sammenligning.
Vi sier ikke at tallTxt er lik hemmeligTall, vi spør om tallTxt er lik hemmeligTall.
Nå sier vi hva som skal skje om vi gjetter riktig.
Mellom parentesene til if kan vi bare skrive en kort tilbakemelding med showMessageDialog():

if(tallTxt == hemmeligTall){

	showMessageDialog(null, "Du gjettet rett!");

}

Programmet burde se omtrent slik ut nå:

package pakke;

import static javax.swing.JOptionPane.*;
import static java.lang.Integer.*;

public class GjettTall { 

	public static void main(String[] args) {
	
		String tallTxt = showInputDialog("Gjett tallet:");
		int tall = parseInt(tallTxt);
		int hemmeligTall = 59;
		
		if(tall == hemmeligTall){

			showMessageDialog(null, "Du gjettet rett!");

		}
		
	}

}

Hvis du kjører dette programmet vil et riktig resultat bli anerkjent, mens et feil resultat vil bli ignorert.
Det er her "else if" kommer inn. Vi kan skrive en liten "else if" som spør om tallet er større enn det hemmelige tallet.
Denne burde komme etter slutten av if, dvs. etter høyre krøllparentes.

else if(tall > hemmeligTall){

	showMessageDialog(null, "tallet du gjettet er for stort");

}

Her ser vi at den er ganske lik den vanlige if.
Vi bruker større enn tegnet for å teste om tall er større enn hemmeligTall.
Med denne blokken vil brukeren vite om tallet er for stort.
Vi kan også lage en til en ny "else if" for å se om tallet er mindre enn hemmeligTall.
Denne setter vi igjen etter slutten av den forrige "else if":

else if (tall < hemmeligTall) {

	showMessageDialog(null, "Tallet du gjettet er for lite");
	
}

Nå vil du ha et program som lar deg gjette et tall og gir tilbakemelding om det enten er rett, for stort eller for lite.
Istedenfor den siste "else if" kunne du også brukt else.
else sier bare hva som skal skje i alle andre tilfeller, og skrives etter alle if og "else if" for å dekke resten av tilfellene.
