Objekter

Java er et objekt-orientert språk. Det finnes en type kalt Object, men det er ikke det i seg selv vi skal fokusere på.
Vi skal fokusere på objekter man kan lage selv. Disse objektene vil arve egenskaper fra Object klassen. Det er også innebygde typer som er objekter, som f. eks. tabeller.
Hva skal vi lagre i et objekt? Hva du kan lagre i et objekt er ganske fleksibelt. Det vil være en egen klasse, med egne variabler og metoder.
Det vil si at de båre kan lagre mange forskjellige typer informasjon, i tillegg til at de kan gjøre mange forskjellige handlinger som vi selv kan definere.
I dette eksempelet skal vi ha et objekt kalt nettside som holder på informasjon om en nettside og lar oss sammenligne slik informasjon mellom flere nettside objekter.
Først lager vi et skjellet for en klasse vi kaller NettsideKlient. Denne vil lage nye Nettside objekter og bruke metodene til de objektene.
Vi skal også lage en Nettside klasse i en annen fil, men denne vil ikke bruke en main metode. Vi trenger også ikke å gjøre klassen "public", og mer privat er generelt bedre, så vi benytter muligheten.

class Nettside {

}

Inne i klassen vil vi ha noen variabler, disse skal brukes innenfor samme klasse, så vi lar de være private:

private String navn;
private String URL;
private String ipAdresse;

Dette er generel informasjon som hvert Nettside objekt vil inneholde.
Nå bruker vi noe kalt en "kontruktør". Den må ha samme navn som klassen og lar deg ta inn parametere når en ny kopi av objektet lages.

public Nettside(String _navn, String _URL, String _ipAdresse){
		
		navn = _navn;
		URL = _URL;
		ipAdresse = _ipAdresse;
		
}

Her vil de private variablene bli satt til argumenter slik at vi kan lage nye objekter som øyblikkelig er meningsfulle.
Vi kan også lage forskjellige versjoner av denne konstruktøren. Det vi gjør nå er å lage en versjon med null argumenter i tilfellet det ikke blir gitt.
Dette vil da være standardverdiene:

public Nettside() {

		this("hjemme", "127.0.0.1", "127.0.0.1");
		
}

Her bruker vi også this. "this" er det vi er i, som er Objektet nettside nå. Vi kaller den forrige konstruktøren som tar 3 paremetre og gir 3 standard parametre.
Nå skal vi også generere noen metoder, set og get, for variablene vi har. Plasser den blinkende streken som viser hvor du er etter slutten av den siste konstruktøren.
Det er her de genererte metodene vil bli plassert. Klikk på Source > Generate getters and setters i topp-baren. Velg alle variablene. Generer.

Nå vil vi ha noen metoder som lar oss endre og lese private variabler utenfor klassen. Generelt vil du ha private variabler og åpne hull til endringer vi tillater med offentlige metoder.
Metoder som bare brukes innenfor samme klasse skal også være private.

La oss lage noen Nettside objekter i NettsideKlient for å teste disse metodene:

Nettside youtube = new Nettside("YouTube", "https://www.youtube.com/", "172.217.21.174");
Nettside wikipedia = new Nettside("Wikipedia", "https://www.wikipedia.org/", "91.198.174.192");
Nettside w3schools = new Nettside("w3schools", "https://www.w3schools.com/", "192.229.133.221");

Her er "new Nettside()" delen som gir parametre til kontruktøren.

Du kan teste metodene ved å skrive f. eks:

System.out.println(youtube.getNavn());

For å printe ut innholdet i variabelen navn i Nettside objektet youtube.

La oss lage noen sammenlignings metoder:

public boolean likNavn(Nettside annenNettside) {
		
	if (annenNettside == null) {
			
		return false;
			
	} else if (this.navn == annenNettside.navn) {
			
		return true;
			
	} else {
			
		return false;
			
	}
		
}
	
public boolean likURL(Nettside annenNettside) {
		
	if (annenNettside == null) {
			
		return false;
			
	} else if (this.URL == annenNettside.URL) {
			
		return true;
			
	} else {
			
		return false;
			
	}
		
}
	
public boolean likIP(Nettside annenNettside) {
		
	if (annenNettside == null) {
			
		return false;
			
	} else if (this.ipAdresse == annenNettside.ipAdresse) {
			
		return true;
			
	} else {
			
		return false;
			
	}
		
}

Her har vi en sjekk for andre objekter som er lik null siden det ville gitt en feil om vi behandlet de.
Du har kanskje lagt merke til at vi ikke bruker en get-metode her for å få tak i variablene til det andre objektet som er paremetret.
Dette går hvis paremetret er av samme type, det er sant for annenNettside siden den er av typen Nettside.

Du kan kalle disse metodene fra NettsideKlient slik som dette:

System.out.println(youtube.likURL(wikipedia));

Dette vil printe ut sannhetverdien, som burde være false i dette tilfellet.

Noe annet du også kan gjøre er å generere en toString() metode slik som med setters og getters.
Hvis du gjør dette vil du også se at det begynner med:

@Override

Dette betyr at vi overskriver en metode som det allerede finnes.
Dette vil gi en String som er en oppsummering av variablene i objektet.
Du kan lage hvilke som helst andre metoder og variabler for dette objektet, det er det som gjør de så nyttige.
