Enkel Interaksjon

Med et bibliotek som heter JOptionPane kan vi lage enkel interaksjon mellom en bruker og programmet.
Etter vi har laget et skjellet, den grunnleggende koden, må vi importere JOptionPane.
Dette gjør vi ved å skrive dette etter package linjen:

import static javax.swing.JOptionPane.*;

Da vil vi ha dette: 

package pakke;

import static javax.swing.JOptionPane.*;

public class HelloWorld { 

	public static void main(String[] args) {

	}

}

Dette gjør at vi kan bruke alle metodene i biblioteket JOptionPane.
Og dette kan vi bruke til å lage en ny verisjon av HelloWorld, som inkluderer navnet til brukeren.
Den første metoden fra JOptionPane vi vil bruke da er showInputDialog().
showInputDialog() tar en melding til brukeren(en String) som parameter.
Verdien som showInputDialog returnerer er også en String verdi.
Det vil si at vi kan sette en String til å være lik en showInputDialog().
Da vil Java kj?re denne metoden n?r den skal gi en verdi til variabelen.
Vi gjør dette i main metoden, slik som dette: 

String navnTxt = showInputDialog("Gi et navn");

Hvis du kjører programmet nå burde det komme opp et lite vindu.
Vinduet sier det du skrev inn som parameter, og returnerer teksten du skriver inn når du klikker "OK".
Da vil navnTxt bli gitt denne verdien.

Nå sier ikke programmet noe etter du har gitt et navn.
Vi kunne ha fikset dette med System.out.println(), men vi skal bruke en til metode fra JOptionPane.
showMessageDialog() lar deg gi en melding til en bruker.
Hvorfor skal vi bruke dette istedenfor System.out.println()?
Vi bruker showMessageDialog() siden en bruker kan se meldingen.
Før vi setter inn noe er det viktig å vite at showInputDialog trenger to parametre.
Parametere er separert med komma og må alltid være av riktig type og i riktig rekkefølge.
Som det første parameteret for showMessageDialog vil vi gi verdien null.
Dette er en spesiel tom variabel som betyr at det ikke er noen verdi, det er ikke det samme som et tall 0.
Vi bruker et komma som en separator og gir en String verdi.
Nå begynner vi med "Hello, ", men dette sier ikke noe navn.
Etter det høyre hermetegnet skriver vi et plusstegn og navnTxt for inneholdet i den variabelen.
Etter vi har endet linjen vil den se slik ut:

showMessageDialog(null, "Hello, " + navnTxt);

Nå ser det litt rart ut i det andre parameteret.
Vi skriver et mellomrom etter komma siden det ikke er noen andre mellomrom.
Plusstegnet legger sammen tekst hvis du gir skriver en String verdi, som parameteret er.
Plusstegnet legger ikke til noe mellomrom, og vil behandle tall som ren tekst.
Dette vil si at 2 + 5 i en String vil gi verdien 25, ikke 7, med mindre du skriver (2 + 7).
Variabeler, som navnTxt, skal vi som sagt ikke legge hermetegn rundt,
siden det er et navn som refererer til en verdi med hermetegn inkludert.

Nå burde programmet se slik ut:

package pakke;

import static javax.swing.JOptionPane.*;

public class HelloWorld { 

	public static void main(String[] args) {
	
		String navnTxt = showInputDialog("Gi et navn");
		showMessageDialog(null, "Hello, " + navnTxt);

	}

}

Hvis du kjører programmet nå, burde det spør det om navn og hilse deg.
